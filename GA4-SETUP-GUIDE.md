# Google Analytics 4 设置指南

## 📋 概述

您的网站已经完全集成了Google Analytics 4（GA4），只需要获取您的测量ID并进行最后配置即可开始收集数据。

## 🚀 快速设置步骤

### 1. 创建GA4账户和属性

1. **访问 Google Analytics**
   - 打开 [analytics.google.com](https://analytics.google.com)
   - 使用您的Google账户登录

2. **创建账户**
   - 点击"开始测量"
   - 输入账户名称：`Sound Healing App` 或 `声音疗愈`
   - 选择数据共享设置（建议全选以获得更好的洞察）

3. **创建属性**
   - 属性名称：`soundflows.app`
   - 时区：选择您的时区
   - 货币：选择相关货币（如USD或CNY）

4. **选择业务信息**
   - 行业类别：`健康与保健` 或 `媒体与娱乐`
   - 业务规模：选择适合的规模
   - 使用意图：选择"测量网站流量"等相关选项

5. **创建数据流**
   - 选择"网站"
   - 网站URL：`https://soundflows.app`
   - 流名称：`Sound Healing Website`

### 2. 获取测量ID

创建完成后，您会看到类似 `G-XXXXXXXXXX` 的测量ID。

### 3. 更新网站配置

将测量ID替换到以下文件中：

#### 文件 1: `index.html`
```javascript
// 在第1077行附近，将G-XXXXXXXXXX替换为您的实际测量ID
<script async src="https://www.googletagmanager.com/gtag/js?id=G-YOUR-ACTUAL-ID"></script>
```

#### 文件 2: `assets/js/ga4-config.js`
```javascript
// 在第11行，将G-XXXXXXXXXX替换为您的实际测量ID
this.measurementId = 'G-YOUR-ACTUAL-ID';
```

### 4. 测试配置

1. **部署到生产环境**
   ```bash
   git add .
   git commit -m "Add Google Analytics 4 integration"
   git push origin main
   ```

2. **验证数据收集**
   - 访问您的网站 https://soundflows.app
   - 在GA4中查看"实时"报告
   - 应该能看到您的访问数据

## 🎯 关键指标仪表板

建议在GA4中创建以下自定义报告：

### 1. 音频使用报告
- **指标**: 音频播放次数、播放时长、完成率
- **维度**: 音频类别、用户细分

### 2. 用户参与报告
- **指标**: 会话时长、页面停留时间、滚动深度
- **维度**: 设备类型、地理位置

### 3. 转化漏斗
- **事件序列**: 访问 → 浏览分类 → 播放音频 → 完成会话

## 🔧 进阶配置

### 1. 目标设置
在GA4中设置以下转化事件：
- `audio_session_complete` - 完成一个音频会话
- `audio_play` - 播放音频（微转化）
- `time_on_page` - 停留超过3分钟

### 2. 受众群体创建
- **活跃用户**: 近7天内播放过音频
- **睡眠用户**: 主要在晚间使用
- **冥想用户**: 主要播放冥想/瑜伽内容
- **高价值用户**: 会话时长超过10分钟

### 3. 数据导出
设置数据导出到Google Sheets或BigQuery以进行深度分析。

## 🛠️ 故障排除

### 常见问题

1. **数据未显示**
   - 检查测量ID是否正确
   - 确认网站已部署最新版本
   - 查看浏览器控制台是否有错误

2. **事件未触发**
   - 确认gtag加载完成
   - 检查自定义事件代码
   - 使用GA4调试模式

3. **实时数据延迟**
   - GA4实时报告可能有1-2分钟延迟
   - 标准报告需要24-48小时处理

### 调试模式

开发环境会自动启用调试模式，您可以在浏览器控制台看到详细的GA4事件日志。
